# BUG-003: Admin não consegue aprovar chamado pendente

## Informações
- **Data:** 15/01/2026
- **Teste:** ADM-016
- **Usuário Testado:** Administrador (admin@garageinn.com.br)
- **Executor:** GPT-5.2 Codex

## Comportamento Esperado
Admin deve ter opção de **aprovar** chamados pendentes.

## Comportamento Atual
Na tela do chamado pendente de aprovação, só existe o botão **Negar**. Não há botão **Aprovar**.

## Passos para Reproduzir
1. Fazer login como admin.
2. Acessar `Chamados` → abrir um chamado em **Aguardando Aprovação**.
3. Verificar a área de ações.

## Evidências
- Screenshot: `.playwright-mcp/bug-003-adm-016-aprovar-chamado-sem-botao.png`

## Observações
Chamado #15 (Compra de DDD) em aprovação nível Encarregado.

---

## Resolução

**Status:** ✅ Corrigido  
**Data:** 17/01/2026

### Causa Raiz

O componente `ticket-approvals.tsx` não exibia os botões de Aprovar/Rejeitar para administradores nos chamados de Compras e Manutenção.

### Correção Aplicada

Os componentes de aprovação foram atualizados para incluir botões de Aprovar e Rejeitar visíveis para usuários com permissão `admin:all`:

- `apps/web/src/app/(app)/chamados/compras/[ticketId]/components/ticket-approvals.tsx`
- `apps/web/src/app/(app)/chamados/manutencao/[ticketId]/components/ticket-approvals.tsx`
- `apps/web/src/app/(app)/chamados/rh/[ticketId]/components/rh-ticket-approvals.tsx`
- `apps/web/src/app/(app)/chamados/sinistros/[ticketId]/components/claim-approvals.tsx`

### Validação

Testado em 17/01/2026:
- Login como admin@garageinn.com.br
- Acessar Chamados → Chamado #15 "Compra de DDD"
- Botões "Aprovar" e "Rejeitar" visíveis na seção de Aprovações
